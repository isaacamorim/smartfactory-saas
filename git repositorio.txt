seu PC local:
git status

E no servidor:
cd /opt/smartfactory-saas
git log --oneline -5
git status
Assim consigo ver exatamente o que está diferente entre local e servidor. Mas enquanto isso, o fluxo correto é esse:

No seu PC local (quando terminar de editar):
cd smartfactory-saas

# Ver o que mudou
git status

# Adicionar tudo
git add .

# Commitar com mensagem
git commit -m "atualização dos arquivos"

# Enviar para o GitHub
git push origin main

No servidor (para puxar o que foi enviado):
cd /opt/smartfactory-saas

# Puxar as atualizações do GitHub
git pull origin main

Se der conflito no servidor, faça reset forçado (descarta o que estava no servidor e pega tudo do GitHub):
cd /opt/smartfactory-saas
git fetch origin
git reset --hard origin/main

Se o repositório no servidor não tem o .git (foi copiado manual em vez de clonado), aí precisa recriar:
cd /opt
rm -rf smartfactory-saas
git clone https://github.com/isaacamorim/smartfactory-saas.git


Veja os arquivos:

ls

Agora devem aparecer os arquivos novos que você subiu.
